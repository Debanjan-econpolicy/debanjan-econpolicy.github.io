### Session 4 — Google Search Console SEO fixes
Diagnosed why site was not appearing in Google Search. Key findings and fixes:

1. **Homepage missing from sitemap (root cause)** — `index.html` had no Jekyll frontmatter, so the `jekyll-sitemap` plugin treated it as a static file and excluded it from `/sitemap.xml`. Google's crawler relies on the sitemap to discover pages.
2. **Fix — `_layouts/none.html` created** — A one-line layout that outputs `{{ content }}` unchanged. This lets Jekyll process `index.html` (so it enters the sitemap) without wrapping it in any theme.
3. **Fix — frontmatter added to `index.html`** — Set `layout: none`, `sitemap: true`, plus proper `<meta name="description">`, canonical URL, and Open Graph tags.
4. **Fix — SEO meta tags improved in `custom-page.html`** — Added canonical URL, `og:title`, `og:description`, `og:url` to all inner pages.

**Incident — homepage broken (immediately fixed):** The first attempt added frontmatter without setting `layout:`. Jekyll's `_config.yml` defaults apply `layout: single` (minimal-mistakes theme) to all pages without an explicit layout, so the homepage was briefly wrapped in the old theme. Fixed by creating `_layouts/none.html` and setting `layout: none`.
