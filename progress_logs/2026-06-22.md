### Session 4 ‚Äî Google Search Console SEO fixes
Diagnosed why site was not appearing in Google Search. Key findings and fixes:

1. **Homepage missing from sitemap (root cause)** ‚Äî `index.html` had no Jekyll frontmatter, so the `jekyll-sitemap` plugin treated it as a static file and excluded it from `/sitemap.xml`. Google's crawler relies on the sitemap to discover pages.
2. **Fix ‚Äî `_layouts/none.html` created** ‚Äî A one-line layout that outputs `{{ content }}` unchanged. This lets Jekyll process `index.html` (so it enters the sitemap) without wrapping it in any theme.
3. **Fix ‚Äî frontmatter added to `index.html`** ‚Äî Set `layout: none`, `sitemap: true`, plus proper `<meta name="description">`, canonical URL, and Open Graph tags.
4. **Fix ‚Äî SEO meta tags improved in `custom-page.html`** ‚Äî Added canonical URL, `og:title`, `og:description`, `og:url` to all inner pages.

**Incident ‚Äî homepage broken (immediately fixed):** The first attempt added frontmatter without setting `layout:`. Jekyll's `_config.yml` defaults apply `layout: single` (minimal-mistakes theme) to all pages without an explicit layout, so the homepage was briefly wrapped in the old theme. Fixed by creating `_layouts/none.html` and setting `layout: none`.

### Session 6 ‚Äî JSON-LD Person structured data added

Added `<script type="application/ld+json">` block to `index.html` with full Schema.org `Person` markup. This tells Google explicitly who the page is about without having to infer it from page text.

Fields included:
- `name`, `jobTitle`, `description` ‚Äî for name and role searches
- `affiliation` ‚Äî IRMA and IIMA (`CollegeOrUniversity` type) ‚Äî for institution searches
- `alumniOf` ‚Äî ICAR-NDRI
- `knowsAbout` ‚Äî 12 research topics (Development Economics, Causal Inference, Field Experiments, Impact Evaluation, etc.) ‚Äî for topic searches
- `sameAs` ‚Äî LinkedIn, Twitter, GitHub ‚Äî connects existing profiles to the website

This is the final technical SEO improvement. All remaining gains now depend on off-page actions (see pending work below).
